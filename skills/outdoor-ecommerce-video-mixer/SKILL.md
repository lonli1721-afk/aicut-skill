# 户外电商视频混剪

## 核心原则

直播口播是主叙事，产品实拍视频和产品照片是画面增强。不要为了凑时长切断一句话，不要让字幕和语音不一致，不要在同一个草稿里重复使用同一段素材。每次生成多个版本时，口播内容、素材顺序、卖点角度都要有明显差异，降低同质化。

最重要：不要先按时间段随便拼。必须先从 ASR 词级时间轴里整理“可用口播候选段”，给每段标注主题、完整度、是否含气口、是否适合作为结尾，再组合成草稿。没有完成这一步，不要调用 VectCut 生成草稿。

## 必要输入

- 产品信息：产品名、品牌、类目、目标市场、语言、卖点、痛点、使用场景、价格和优惠是否展示。
- 素材信息：直播原片或直播切片、产品实拍视频、产品图、BGM、音效；本地路径和已上传公网 URL 都要记录。
- 视频规格：平台、版本数、时长范围、画幅、语言、是否保留直播原声、是否加字幕、是否加 BGM。
- 接口配置：火山引擎豆包 ASR 的 API Key 或 Access Token、Resource ID、接口模式；VectCut API Base URL 和 API Key。
- 交付方式：只生成请求、直接创建 VectCut/剪映草稿，还是同时生成本地预览。

## 默认工作流

1. 建立素材 manifest，记录每个素材的来源、用途、可用时长、是否允许声音、是否允许重复。
2. 用火山引擎豆包流式语音识别模型 2.0 转录直播口播，优先使用双向流式模式；保存原始响应、句子表和词级时间轴。
3. 先输出并审查“口播候选段表”：每段必须包含 `start/end`、原文、主题角度、是否有气口、是否完整、是否适合结尾。剔除含明显气口、半句话、纯互动、价格凑单教程、没有商品价值的闲聊段。
4. 从最终词级时间轴选择口播片段，按完整语义切分；每条草稿最后一个片段必须是完整闭合卖点句，不允许用“如果/只要/我给大家/为什么/然后/那个/嗯/你看”这类引出下一句的口播收尾。
5. 生成多个版本时，为每个版本选择不同主卖点、开头、素材顺序和画面组合，控制版本之间的口播与画面重合度。
6. 四条草稿必须先写清楚四个不同角度，例如：品牌背书、抓地防护、脚感掌控、场景价格。每条只围绕一个主角度展开，最多补 1-2 个辅助卖点，不要把所有卖点平均塞进每条。
7. 用词级时间轴生成小段字幕；字幕必须来自直播口播原文，不要用产品卖点文案替代真实口播。
8. 用 VectCut 创建剪映草稿；每次不要只看 `create_draft`，要检查每个操作的 `success/error`。
9. 质检草稿：有无黑屏、无画面、两层直播声音、字幕错位、图片停留过长、结尾话没说完、素材重复、音频丢失。

## 混剪规则

- 直播口播是主音轨，BGM 是低音量底音；产品实拍视频默认静音，除非用户明确要求保留原声。
- 如果画面使用直播原画面，可以直接保留该直播片段声音；不要再额外叠一条同内容直播音轨，避免双重声音。
- 如果画面使用产品图或实拍 B-roll，同时仍要有直播口播；不要让图片或 B-roll 单独无口播停留太久。
- 图片每张在单个草稿里最多出现一次，每次 2-3 秒左右，并且必须压在正在说话的口播上。
- 画面素材必须首尾紧密衔接，不能出现黑屏、空画面、画面断档或素材之间留白。无论直播画面、B-roll、图片怎么穿插，时间线上必须始终有可见画面覆盖。
- 颜色、上脚、实物展示类口播优先使用直播原场景画面，因为观众需要看到主播展示的鞋和颜色。
- 单个草稿内部素材不要重复；多个草稿之间也要降低同一句口播、同一实拍段、同一图片的重合度。
- 结尾不要用“如果你……”这类未闭合句式；优先用明确卖点收尾，例如快干、透气、抓地、防护、通勤户外都能穿。

## 气口和口播剪辑

- 尽量去掉“额”“嗯”“emmm”“那个”“就是说”等明显气口、犹豫和思考声。
- 不要为了去气口把正常句子剪得断断续续；完整语义优先于机械静音检测。
- 遇到“我来给你们看一下”“为什么这么说”这类明显引出下一句的片段，不要当结尾。
- 如果 ASR 文本明显和原视频不一致，先复核或重新转录，不要用错误文本继续生成字幕。
- 气口处理优先级：先从候选段层面剔除整段气口密集的片段；其次只在词级时间轴里删除独立的“额、嗯、emmm、那个”；不要只按静音波形硬剪，因为会把正常句子剪碎。
- 如果一个片段开头或中间出现 `嗯/额/那个`，必须确认删除后句子仍然通顺；不通顺就整段不用。
- 禁止把带有明显气口的段落作为开头或结尾。

## 四条差异化生成规则

- 每条成片默认控制在 30 秒左右，通常接受 28-36 秒；除非用户明确说“不要求 30 秒”，否则不要做成 20 秒出头或过长。
- 四条视频的主角度必须不同，先写出角度再选片段，不允许先选片段后硬凑角度。
- 每条开头 3-5 秒必须进入明确卖点，不要用无意义寒暄或链接提示开头。
- 每条结尾必须落在完整的商品价值判断上，例如“抓地力天花板”“支撑性非常到位”“50 公里以内完全没问题”“性价比多高了”。不能以“我给大家”“你看”“如果您”“只要您想要去”收尾。
- 版本之间口播原文重合度尽量低；同一句强卖点最多出现在一条主版本里，其他版本换角度表达。
- 版本之间素材顺序也要变化。不要四条都采用“直播开头+B-roll 中段+点击下单”的同一结构。
- 如果某条必须用同一直播片段作为核心卖点，其他条不要再用同一时间段的画面或音频。

## 字幕规则

- 口播字幕完全根据直播语音生成，不要凭产品文案补字幕。
- 字幕按词级时间轴切成小段，通常 0.5-1.3 秒一段；长度尽量适中，优先控制在 5-8 个中文字符左右，避免剪映花字模板因文本过长自动缩小。切点不要拆开固定搭配和序数短语，例如 `第二个`、`第三款`、`第一双` 要尽量同屏。
- 不要乱拆词：`Chaco`、`ODYSSEY`、`溯溪鞋`、`溯溪凉鞋`、`户外凉鞋`、`鞋面`、`鞋头`、`鞋底`、`鞋床`、`防穿刺`、`防滑`、`透气`、`速干`、`Z字绑带`、`棋盘格`、`loveset鞋床`、`重装徒步`、`全地形` 等词尽量保持同屏。
- 不要机械按字数硬切。字幕不要以 `的、了、着、过、呢、吗、吧` 等虚词开头；也不要以 `去、穿、看、拿、换、买、做、当、用、搭、踩、走、跑、选、放` 等明显需要宾语的动词结尾。遇到 `跑山、徒步、溯溪、上脚、海军蓝、运动员、第二个` 这类自然词组，要尽量同屏。
- 避免单独出现 1-2 个字的短字幕，尤其是“的”“了”“是”这种虚词；合并到前后字幕里。
- 字幕尽量去掉逗号、句号、空格和明显口头气口，但不要删除会改变意思的词。
- 如果使用花字模板，同样字号在剪映里可能因为文本长度自动缩放；要通过控制每段字幕长度来维持视觉统一。

## 当前偏好样式

适用于 ODYSSEY/Chaco 溯溪鞋这类草稿：

- 口播字幕：花字 `W0BtRFRVQlRAa19XSFpBa0tWUQ==`，字体 `CC_Captial`，字号 11，缩放 100%，位置约 `transform_x=0.0, transform_y=-0.40`，不要在口播字幕内部拆出高亮词。
- 卖点字幕：可以使用花字 `WkpuRFxSQVNFZlhRSV1Ob0FVVA==` 或用户指定的新模板；卖点不要和对号贴纸重叠，字体 `CC_Captial`，字号 9，缩放 100%，位置约 `transform_x=0.32, transform_y=-0.10`，整体在口播字幕右上方。
- 重点关键词字幕必须保留：每条至少 1-3 个，放在画面上方或不遮挡主体的位置，用用户指定的关键词文字模板轮换；关键词要根据当前口播内容总结，并且必须跟随对应口播卖点的时间点出现，不要提前、滞后或随机摆放。不要和同一时间出现的卖点字幕重复。关键词适合写成“抓地很稳”“防护到位”“跑山好控”“性价比很高”这类短促标题，不要写“重点”两个字。
- 卖点出现时可以用预设 `audio3`；点击下单用 `audio7`；关键词音效可用 `audio2/audio5/audio6` 混合，但音效素材如果本地丢失频繁，可以先去掉音效。
- “点击下单”可以保留，但不要加过丑的红底；如果用户嫌乱，先删掉额外凸显词，只保留口播字幕和必要卖点。
- 不要自动做封面首帧。封面如果需要，单独让用户确认候选画面或使用剪映手动设封面。

## VectCut 注意事项

- `/cut_jianying/add_video` 的 `track_name` 不是剪映物理主轨；即使叫 `video_main`，也可能在剪映里显示为普通视频轨。不要为了手机设封面反复折腾物理主轨，除非改走本地剪映草稿 JSON 或用户提供主轨占位预设。
- VectCut 不允许同一视频轨上有重叠片段；如果把 B-roll 强行放到同一个 `track_name`，可能报 `New segment overlaps with existing segment`。
- 稳定策略：直播片段作为主要可见视频且保留口播；B-roll/图片作为上层覆盖，B-roll 静音。这样不容易出现黑屏或双重声音。
- 如果出现黑屏、无画面或接口成功但草稿内容为空，先检查是否有重叠视频轨、隐藏视频轨、过度缩放、或图片/视频动画导致渲染失败。
- 长直播 URL 的 `start/end` 截取可能不稳定；如果字幕和口播完全对不上，先用 FFmpeg 预切小片段并上传，再用小片段 URL 创建草稿。
- VectCut 预设音频可能在剪映本地弹“链接媒体”。如果用户不想手动补链，减少或去掉 `audio2/audio3/audio5/audio6/audio7` 这类短音效，BGM 优先使用用户自己上传的稳定链接。
- VectCut 素材库容量按订阅计划变化；免费约 200M，基础约 2G，标准约 10G，高级约 100G。扩容通常是升级订阅，不是 API 参数。
- API Key 不要写入 skill、日志或聊天记录；通过环境变量或 PowerShell 安全输入传入。

## 质检清单

生成后必须检查：

- 所有 VectCut 操作 `Fails: 0`，不要只看草稿链接是否生成。
- 每条草稿必须列出最终使用的口播 `source_range` 和最后一句原文；确认最后一句已说完。
- 每条草稿时长是否在 30 秒左右；除非用户放宽要求，否则 28-36 秒优先。
- 每条草稿必须列出主角度；四条主角度不能重复。
- 每条是否有重点关键词字幕，且关键词没有和卖点字幕同屏重复、没有遮挡主体。
- 重点关键词的出现时间是否贴合当前口播内容；如果口播还没讲到该卖点，关键词不能先出现，如果口播已经换话题，关键词必须及时消失或换成对应内容。
- 口播字幕是否和真实语音对齐，是否有“鞋/面”“鞋/头”这类断词。
- 是否还有“额、嗯、emmm、那个”等明显气口。
- 是否出现双重直播声音，尤其是直播画面加隐藏口播音轨的场景。
- 是否有黑屏、空画面、素材不挨着、时间线留白、图片停太久、B-roll 原声、素材重复。
- 结尾是否是一句完整卖点话，不是半句话或引出下一句。
- 多个草稿之间素材和口播重合度是否过高。
