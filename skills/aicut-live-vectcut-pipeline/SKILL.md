# 直播剪口播到 VectCut 批量混剪

## 总原则

先把直播口播清洗成可靠母版，再进入批量混剪。不要跳过审核页直接混剪；不要用错误 ASR 文本生成字幕；不要为了去气口把完整句子剪碎；不要生成有黑屏、双重直播声音、素材留白或半句话结尾的草稿。

本技能融合两个阶段：

1. **AI剪口播阶段**：直播视频转录 -> 识别气口、口误、停顿、重复 -> 打开审核页面 -> 用户确认删除 -> 生成干净口播母版。
2. **aicut/VectCut 阶段**：基于干净口播母版和产品素材，按不同角度批量生成低同质化剪映草稿。

每次用户指出问题，都要判断是“单次修正”还是“长期偏好”。长期偏好写入学习记录，下次同类产品自动套用。

## 必要输入

- 直播素材：本地路径或公网 URL。用于转录、审核、生成干净口播母版。
- 产品信息：产品名、品牌、类目、目标平台、语言、卖点、痛点、使用场景、价格是否展示。
- B-roll 素材：户外实拍视频、产品图、细节图、可用时长、是否允许原声。默认 B-roll 静音。
- 音频素材：BGM、短音效、预设音效编号。BGM 低音量，口播是主音轨。
- VectCut 配置：`VECTCUT_API_BASE_URL`、`VECTCUT_API_KEY`、`preset_id`、文字模板/花字模板/音效模板。
- 生成规格：版本数量、是否尽可能多生成、平台、画幅、时长范围、是否字幕、是否 BGM、是否点击下单。

如果用户没有指定，默认：

- 每个产品至少生成 4 条；如果用户说“更多条/尽可能多/最多”，先评估素材和口播容量，再生成尽可能多的低同质化版本。
- 抖音/短视频竖屏 9:16。
- 每条 30 秒左右，优先 28-36 秒。
- 中文口播字幕。
- 保留直播口播，B-roll 静音，BGM 低音量。
- 先生成草稿，不直接导出成片。

## 阶段 1：直播转录

优先调用火山引擎豆包语音识别，保存原始响应、词级时间轴和句子映射。若 ASR 明显和原视频不一致，必须重新转录或人工复核，不要继续生成字幕。

转录结果必须整理成：

- `words.json`：每个词的 `text/start/end/confidence`。
- `sentence_map.json`：每句话的 `sentence_id/start/end/text/word_indices`。
- `raw_asr.json`：原始接口响应，便于追溯。

识别后先做文本清洗，但不能直接删除有语义的词。清洗范围只包括明显气口、停顿词和口误候选，例如：

- `额`
- `嗯`
- `emmm`
- `那个`
- `就是说`
- 明显拖长的无意义语气词
- 重复开头、说错后立刻纠正的前半句

## 阶段 2：审核页和干净口播母版

必须生成审核页面，让用户确认哪些片段删除。审核页应展示：

- 原视频或音频预览。
- 每句口播文本。
- AI 建议删除的气口、停顿、口误、重复。
- 删除后剩余语义是否完整。
- 可取消勾选或手动新增删除段。

审核完成后，根据 `review_log.json` 生成干净口播母版：

- 输出 `clean_voice_master.mp4`。
- 输出 `clean_voice_master_audio.wav`。
- 输出 `clean_sentence_map.json`，保留清洗后每句话的时间轴。

干净母版是后续混剪的唯一口播来源。进入 VectCut 前，必须确认：

- 没有明显 `额/嗯/emmm/那个` 等气口。
- 不存在一句话中间被剪断。
- 不存在“我给大家看一下”“为什么这么说”“如果你”这类引出下一句的片段作为结尾。
- 每个可用片段都有完整语义和对应时间轴。

## 阶段 3：候选段表

从干净母版整理“口播候选段表”，不要直接按时间硬切。每段包含：

```json
{
  "id": "seg-001",
  "start": 0.0,
  "end": 6.5,
  "text": "这一双下水走溪边非常稳",
  "topic": "防滑抓地",
  "selling_points": ["防滑", "抓地稳"],
  "is_complete": true,
  "can_open": true,
  "can_close": true,
  "has_filler": false,
  "visual_hint": "直播展示鞋底或户外踩水画面"
}
```

候选段筛选规则：

- 只保留完整语义段。
- 删除或避开气口密集段。
- 删除纯互动、纯寒暄、无商品价值闲聊。
- 颜色、上脚、实物展示类口播优先匹配直播原画面。
- 结尾段必须是完整卖点判断，不能是下一句的引子。

## 阶段 4：多版本差异化混剪计划

每条先确定主角度，再选口播和画面。不要让多条视频使用同一结构、同一开头、同一结尾或同一组核心口播。

默认至少规划 4 个方向；当用户要更多版本时，继续扩展更多角度。方向可按产品自动调整，例如：

- 品牌背书/专业户外品牌。
- 抓地防滑/安全感。
- 包裹支撑/脚感舒适。
- 通勤露营/多场景穿搭。
- 轻量透气/夏季舒适。
- 耐磨防护/复杂路面。
- 颜色穿搭/日常搭配。
- 价格福利/入手理由。
- 人群细分/新手、通勤、露营、跑山、徒步、海边、钓鱼等。

每条计划必须写清：

- 主角度。
- 开头 3-5 秒卖点。
- 使用的口播段 ID。
- 使用的直播画面、B-roll、图片。
- 关键词字幕。
- 卖点字幕。
- 结尾句原文。
- 和其他版本的重合度说明。

重合度规则：

- 同一句核心口播不要在多条里重复使用。
- 同一个 B-roll 片段在单条内不要重复。
- 图片每张单条最多出现一次，每次 2-3 秒，并且必须压在有口播的位置。
- 多条之间素材顺序、口播角度、开头和结尾都要变化。

### 最大可生成数量估算

当用户要求“尽可能多”时，不要直接给固定数量。先根据素材计算一个可交付上限：

- 统计干净口播候选段数量，只计算 `is_complete=true` 且 `has_filler=false` 的段。
- 每条视频优先使用 3-6 个完整口播段，或 28-36 秒左右的连续可用口播。
- 每条至少有 1 个清晰主角度，主角度不能和上一条完全重复。
- 每条至少有 1-3 个与口播匹配的重点关键词。
- 每条内部素材不重复；跨条可以少量复用通用素材，但核心口播和核心画面尽量不重复。
- 如果新增一条会导致核心口播重合度明显过高、画面素材只能重复堆叠、或结尾只能用半句话，就停止生成。

建议输出一个“可生成上限说明”，例如：

```text
本次干净口播可用完整段 38 段，强卖点段 15 段，B-roll 可用 6 条，产品图 3 张。
预计可做 8 条低同质化视频；第 9 条开始会明显复用核心口播，所以本次建议生成 8 条。
```

如果用户明确要求超过建议上限，可以继续生成，但必须标注“会增加素材/口播重合度”。

## 阶段 5：VectCut 草稿生成

用 VectCut 生成剪映草稿时遵守：

- 口播主音轨来自干净母版。
- B-roll 默认静音。
- 画面全程有素材覆盖，不能黑屏、空画面、留白。
- 素材紧密排列。
- 如果使用直播画面且直播画面自带口播，不要再叠同一段口播音轨，避免双重声音。
- 如果使用 B-roll 或图片覆盖直播口播，B-roll/图片不带原声，仍保留干净口播音频。
- 不要把长直播 URL 直接依赖 `start/end` 做精确字幕对齐；如 VectCut 截取不稳定，先预切小片段并上传。

每次调用 VectCut 后检查每个步骤：

- `success` 是否为 true。
- `error` 是否为空。
- 草稿是否真的有画面、音频、字幕。
- 不要只看创建草稿接口是否返回链接。

## 字幕和花字规则

口播字幕必须来自干净母版的真实语音，不能用产品文案替代。

断句规则：

- 每段长度适中，优先 5-8 个中文字符左右。
- 不要拆开固定词组，例如品牌名、产品名、鞋底、鞋面、防穿刺、防滑、跑山、徒步、第二个、第三个。
- 不要让字幕以 `的/了/着/过/呢/吗/吧` 开头。
- 不要让字幕以明显需要宾语的动词结尾，例如 `去/穿/看/拿/换/买/做/当/用/搭/踩/走/跑/选/放`。
- 去掉逗号、句号和明显气口词，但不能改意思。

当前默认样式：

- 口播字幕：花字 `W0BtRFRVQlRAa19XSFpBa0tWUQ==`，字体 `CC_Captial`，字号 11，缩放 100%，位置约 `transform_x=0.0, transform_y=-0.40`。
- 卖点字幕：字体 `CC_Captial`，字号 9，缩放 100%，位置约 `transform_x=0.32, transform_y=-0.10`，不要和对号贴纸重叠。
- 重点关键词：每条 1-3 个，跟随口播内容出现，不要随机提前出现；不要写“重点”两个字；不要和同屏卖点字幕重复。
- 点击下单：如用户要求，放在结尾，使用用户指定模板和 `audio7`。

当用户给出新的模板 ID、花字 ID、字幕位置、字体、字号或音效规则时，本次任务立即覆盖默认值；如果用户多次强调同一规则，写入学习记录。

## 质检清单

生成草稿后必须检查并向用户报告：

- 目标数量的视频是否都生成成功；如果没有生成到用户要求的数量，说明是素材不足、口播不足、接口失败还是低同质化不达标。
- 每条时长是否 30 秒左右。
- 每条主角度是否不同。
- 是否还有 `额/嗯/emmm/那个` 等气口。
- 结尾句是否完整说完。
- 字幕是否对齐真实口播。
- 字幕断句是否自然，没有拆开词组。
- 是否保留重点关键词字幕。
- 重点关键词是否跟随对应口播内容。
- 是否有黑屏、空画面、素材留白。
- 是否有双重直播声音。
- B-roll 是否已静音。
- 单条内部素材是否重复。
- 多条之间素材和口播重合度是否过高。
- VectCut 每个操作是否 `Fails: 0`。

如果任何一项不通过，不要说“完成”。先修正或明确告诉用户是哪一项受限。

## 学习能力

每次用户反馈都要归类：

- **全局偏好**：例如“口播字幕统一 CC_Captial 11 号”“不要凸显词太乱”“重点关键词要跟口播内容”。写入 `accepted_rules.md`。
- **产品偏好**：例如某个品牌名纠错、某类鞋的卖点表达。写入产品学习记录。
- **素材问题**：例如某个 URL 失效、某个音效前半段无声、某个 B-roll 封面不能用。写入素材记录。
- **接口问题**：例如 VectCut 长 URL 截取不稳定、剪映链接媒体、素材库容量不足。写入故障记录。

学习记录格式：

```json
{
  "time": "2026-06-17T12:00:00+08:00",
  "scope": "global|product|asset|api",
  "source_feedback": "用户原话",
  "rule": "下次可执行的规则",
  "applies_to": ["字幕", "气口", "VectCut", "卖点"],
  "confirmed": true
}
```

下次启动任务时，先读历史学习记录，再制定本次计划。不要把用户的一次临时审美意见错误升级成全局规则；如果不确定，先当作本次任务规则。

## 交互节奏

默认不一次性问很多问题。缺少信息时优先用合理默认值，并明确标注假设。只有以下信息缺失时必须问：

- 没有直播素材。
- 没有 VectCut API Key 或无法创建草稿。
- 用户要求直接生成成片但没有可用素材 URL 或本地路径。
- 审核页需要用户确认，但用户还没完成审核。

当审核页生成后，暂停进入混剪，提醒用户先完成审核。用户确认“审核好了/跑完了/继续”后，才使用干净口播母版进行批量混剪。
